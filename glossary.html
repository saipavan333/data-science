data science
Turning data into decisions — combining statistics, programming, and domain knowledge to find and act on patterns.

NumPy's rule for combining arrays of different shapes without writing loops.

A 2-D labelled table (rows and named columns) — the core pandas data structure.
datetime
A value representing a date and time, which you can decompose into calendar features.

A reusable, named block of code that takes inputs and returns an output.

Split rows into groups by a key, apply an aggregation to each, and combine the results.

Reshape a wide table into long form, collapsing many columns into key/value rows.

Combine two tables by matching rows on a shared key (a join, in pandas).

Reshape long data into a wide table, turning distinct values of a column into new columns.
seaborn
A Python plotting library built on matplotlib, tuned for statistical charts.
variable
A named container that holds a value in a program.
vectorization
Doing an operation on a whole array at once (in fast C code) instead of looping element by element.

Statistics53

alternative hypothesis
The claim that there IS an effect — what you accept if the null is rejected.
anova
Analysis of Variance — tests whether the means of three or more groups differ.
base rate
How common something is to begin with — ignoring it leads to badly wrong probability judgements.
bayes' theorem
A rule for updating a probability when you get new evidence, weighting by base rates.
binomial distribution
The distribution of the number of successes in a fixed number of yes/no trials.
bonferroni
A conservative correction for multiple testing: divide your significance threshold by the number of tests.
central limit theorem
The average of many independent samples is approximately normally distributed, whatever the population's shape.
chi-square test
A test of association between categorical variables, comparing observed vs expected counts.
clt
The average of many independent samples is approximately normally distributed, whatever the population's shape.
coefficient of determination
R² — the share of variance in one variable linearly associated with another.
conditional probability
The probability of one event given that another has occurred, P(A|B).
confidence interval
A range from a procedure that would capture the true value in, say, 95% of repeated samples — it expresses uncertainty.
confidence intervals
A range from a procedure that would capture the true value in, say, 95% of repeated samples — it expresses uncertainty.
confidence level
The long-run capture rate of an interval procedure, e.g. 95%.
correlation
A measure of how strongly two variables move together (−1 to +1); not the same as causation.
correlation coefficient
A number from −1 to +1 summarising the strength and direction of a linear relationship.
covariance
How two variables vary together (unstandardized); correlation is covariance scaled to −1..+1.
degrees of freedom
The number of values free to vary in a calculation — it shapes the t and chi-square distributions.
distribution
The full description of how likely each value (or range) of a variable is.
effect size
How big a difference or relationship is — distinct from whether it's statistically significant.
hypothesis test
A procedure that weighs evidence against a null hypothesis to decide if an effect is real.
iqr
Interquartile Range — the spread of the middle 50% of data (Q3 − Q1), robust to outliers.
mean
The arithmetic average; sensitive to outliers.
median
The middle value when data is sorted; robust to outliers.
multiple testing
Running many tests raises the chance of a false positive; correct for it (e.g. Bonferroni).
normal distribution
The symmetric bell curve; many statistics are approximately normal thanks to the CLT.
null hypothesis
The default assumption of 'no effect' or 'no difference' that a test tries to disprove.
one-tailed
A test looking for an effect in one specified direction only.
outlier
A data point far from the rest — sometimes an error, sometimes the most important signal.
outliers
A data point far from the rest — sometimes an error, sometimes the most important signal.
p-value
The probability of seeing data at least this extreme if the null hypothesis were true — small means 'surprising under no effect'.
p-values

The standard correlation coefficient, measuring linear association.

The distribution of counts of rare events in a fixed interval.
population
The entire group you want to draw conclusions about.

The probability a test detects a real effect of a given size — usually targeted at 80%.

A cut point dividing a distribution into equal-probability pieces (e.g. the 90th percentile).

The 25th, 50th, or 75th percentile — the cut points that split data into quarters.
sample
The subset of the population you actually observe.
sampling
Drawing a subset of a population to estimate something about the whole.
significance
Evidence that a result is unlikely to be due to chance alone (typically p < 0.05) — not the same as being large or important.
skew
Asymmetry in a distribution; a long right tail (e.g. income) is right-skew.
spearman
A rank-based correlation that captures any monotonic relationship, not just linear.
standard deviation
The square root of variance — spread in the same units as the data.
standard error
The standard deviation of a statistic (like a sample mean) across repeated samples — how much it would wobble.
statistical significance
Evidence that a result is unlikely to be due to chance alone (typically p < 0.05) — not the same as being large or important.
t-distribution
A bell curve with fatter tails than the normal, used for small-sample inference about a mean.
two-tailed
A test looking for an effect in either direction (the usual default).
type i error
A false positive — concluding there's an effect when there isn't.
type ii error
A false negative — missing a real effect.
underpowered
The probability a test detects a real effect of a given size — usually targeted at 80%.
uniform distribution
A distribution where every value in a range is equally likely.
variance
The average squared distance of values from their mean — a measure of spread.

a/b test
A randomized experiment comparing two variants to measure the causal effect of a change.
control group
The group that does not receive the change, used as the baseline for comparison.
cuped
A variance-reduction technique that uses pre-experiment data to make A/B tests more sensitive.

The smallest effect size an experiment is powered to detect (the MDE).

Trying many analyses and reporting whichever crosses significance — manufacturing false positives.

Repeatedly checking a running test and stopping at the first significant result — it inflates false positives.
primary metric
The single main metric an experiment is designed to move.
randomization
Randomly assigning units to groups so the groups are comparable on everything — the basis of causal inference.
simpson's paradox
A trend that holds in every subgroup can reverse when the groups are pooled together.
srm
Sample Ratio Mismatch — when the actual split (e.g. 50/50) differs from intended, signalling a broken experiment.
two-proportion z-test
A test for whether two conversion rates differ, used to analyse A/B results.
twyman's law
Any figure that looks surprising or too good is probably wrong — check for a bug first.

Causal inference20

average treatment effect
The average causal effect of a treatment across the population (the ATE).
backdoor criterion
The rule for picking variables to adjust for so all confounding backdoor paths are closed.
backdoor path
A non-causal path between treatment and outcome that leaks confounding; close it by adjusting for the right variables.
causal inference
Estimating the effect of a cause from data — what would change if you intervened.
collider
A variable caused by two others; conditioning on it creates a spurious association between them.
colliders
A variable caused by two others; conditioning on it creates a spurious association between them.
confounder
A variable that influences both the treatment and the outcome, creating a misleading correlation.
confounders
A variable that influences both the treatment and the outcome, creating a misleading correlation.
confounding
A variable that influences both the treatment and the outcome, creating a misleading correlation.
counterfactual
What would have happened to the same unit under the opposite treatment — never directly observed.
dag
Directed Acyclic Graph — a diagram of assumed causal relationships used to decide what to control for.
difference-in-differences
Estimating a causal effect by comparing the before/after change in a treated group to that of a control group.
instrumental variable
A variable that nudges the treatment but affects the outcome only through it, used to estimate causal effects.
interference
When one unit's treatment affects another's outcome, breaking the usual independence assumption.
mediator
A variable on the causal path between treatment and outcome; controlling for it removes part of the effect.
mediators
A variable on the causal path between treatment and outcome; controlling for it removes part of the effect.
parallel trends
The assumption behind difference-in-differences: absent treatment, the groups would have moved in parallel.
propensity score
The modelled probability of receiving treatment, used to match comparable treated and control units.
regression discontinuity
Estimating causal effects by comparing units just above vs just below an assignment cutoff.
selection bias
Bias from how the sample was chosen — often a collider in disguise (e.g. only studying admitted patients).

Building models sequentially so each fixes the previous ones' mistakes, reducing bias.

A gradient-boosting library with strong built-in, leakage-safe handling of categorical features.

The mean point of a cluster — k-means moves centroids to the middle of their assigned points.

decision trees

Compressing many correlated features into a few informative axes (e.g. PCA).

Combining many models (by averaging or boosting) to beat any single one.
entropy
A measure of impurity/disorder in a set of labels; trees split to reduce it.
generalize
Perform well on new, unseen data — not just on the data used for training.

An ensemble that adds trees sequentially, each correcting the errors of the last, to reduce bias.

A setting you choose before training (e.g. tree depth, learning rate), not learned from data.

Getting a computer to learn patterns from data instead of being explicitly programmed with rules.
model
A mathematical function, fitted to data, that maps inputs to a prediction.

When a model memorises training noise and fails to generalise (high variance).
pca

An ensemble that averages many independent decision trees to reduce variance (bagging).
random forests

Predicting a continuous number (e.g. price).

Learning from labelled examples to predict a known target (e.g. price, spam-or-not).

underfitting
When a model is too simple to capture the real pattern (high bias).

Finding structure in data that has no labels, such as grouping similar rows (clustering).
xgboost
A fast, powerful gradient-boosting library — a frequent winner on tabular data.

Feature engineering19

categorical
A variable that takes one of a fixed set of categories rather than a number.

When information unavailable at prediction time sneaks into training, inflating scores that then collapse in production.

A single input column a model sees — the model learns from the features you compute, never the raw data.
feature engineering
Turning raw data into informative features a model can use — often where most of the winning happens.
feature selection
Choosing a useful subset of features and dropping noise ones.
features
A single input column a model sees — the model learns from the features you compute, never the raw data.
imputation
Filling in missing values with an estimate (mean, median, or a model), fit on train only.
multicollinearity
When features are highly correlated with each other, making individual coefficients unstable.
nominal
A categorical variable with no inherent order (red, green, blue).
normalization
Rescaling a feature to a fixed range, often 0 to 1.
one-hot encoding
Turning a categorical column into one binary indicator column per category, implying no fake order.
ordinal
A categorical variable with a meaningful order (low < medium < high).
pipeline
A bundled sequence of fitted transforms plus a model, so preprocessing is fit on train only and reused at serving.
pipelines
A bundled sequence of fitted transforms plus a model, so preprocessing is fit on train only and reused at serving.

z-score
Rescaling a feature to mean 0 and standard deviation 1 (a z-score).

Evaluation22

accuracy
The fraction of predictions that are correct — misleading on imbalanced data.
auc
Area under the ROC curve — the probability the model ranks a random positive above a random negative (0.5 = chance).
calibration
Whether predicted probabilities are truthful — of cases called 90% likely, about 90% should occur.
confusion matrix
A 2×2 table of true/false positives and negatives that exposes what accuracy hides.
cross-validation
Splitting data into k folds and training k times so the performance estimate is stable, not one lucky split.
f1 score
The harmonic mean of precision and recall — high only when both are high.
mae
Mean Absolute Error — the average size of prediction errors, robust to outliers.
permutation importance
Measuring a feature's importance by shuffling it and seeing how much performance drops.
pr-auc
Area under the precision–recall curve — the honest summary when positives are rare.
precision
Of everything predicted positive, the fraction that truly is — the cost of false positives.
r-squared
The share of the target's variance a regression explains (1 = perfect, 0 = no better than the mean).
recall
Of all real positives, the fraction the model catches — the cost of false negatives.
rmse
Root Mean Squared Error — like MAE but squares errors first, so it punishes large misses harder.
roc curve
A plot of true-positive vs false-positive rate across all thresholds.
roc-auc
Area under the ROC curve — the probability the model ranks a random positive above a random negative (0.5 = chance).
sensitivity
Of all real positives, the fraction the model catches — the cost of false negatives.
shap
A method that fairly attributes each prediction to its features, giving auditable per-prediction explanations.
stratified
A split/CV that preserves each fold's class balance — needed for imbalanced data.
test set
Held-out data opened once at the end for an honest performance estimate — sacred, never tuned against.
threshold
The probability cutoff that turns a score into a decision; 0.5 is an arbitrary default to tune to costs.
thresholds
The probability cutoff that turns a score into a decision; 0.5 is an arbitrary default to tune to costs.
validation set
Data used to compare models and tune hyperparameters, separate from the final test set.

MLOps12

batch serving
Scoring a pile of data on a schedule and storing results — simple and enough for most use cases.
canary
Routing a small slice of traffic to a new model and watching metrics before full rollout.
concept drift
When the relationship between inputs and the target changes, making the old model wrong.
data drift
When the distribution of incoming features shifts away from what the model was trained on.
latency
The time to serve one prediction — judged by the tail (p95/p99), not the average.
mlops
The practices that make a model run reliably in production — pipelines, versioning, serving, and monitoring.
model registry
A versioned catalogue of trained models with stages and lineage — the source of truth for what's deployed.
online serving
Exposing a model behind an API that scores single requests in milliseconds.
psi
Population Stability Index — a number quantifying how much a feature's distribution has drifted.
serialization
Saving a trained model to a file (e.g. with joblib) so it can be loaded later without retraining.
shadow deployment
Running a new model alongside the old on live traffic without acting on it, to compare safely.
training-serving skew
When preprocessing at serving time differs from training, silently corrupting predictions.

Communication5

bar chart
A chart comparing categories by bar length — start the axis at zero.
bluf

chartjunk
Non-data decoration on a chart (3-D, heavy gridlines, clutter) that obscures the message.
data-ink ratio
The share of a chart's ink devoted to data vs decoration; maximise it by removing chartjunk.
pyramid principle
Communicate answer-first: state the recommendation, then the key arguments, then the supporting detail.
